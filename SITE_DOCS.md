Site documentation — Adam Stirtan

Overview
--------
This repo contains the source and static build for https://www.adamstirtan.net — a small single-page site that renders a terminal-like homepage.

Key locations
-------------
- src/: Vue 3 + Vite source files
  - src/main.ts — app entry
  - src/App.vue — root component
  - src/components/Terminal.vue — the terminal UI; primary place for UI behavior and the "glitch" effect
  - src/style.css — global styles
- public/: static public assets used in dev (if present)
- dist/: production build output (generated by `npm run build`)
- assets/: production assets copied into repo root for simple static hosting (contains built JS/CSS, favicon)
- index.html — production index referencing /assets/* built files

Development
-----------
Requirements:
- Node.js (tested with Node 25+)
- npm

Common commands (run from repo root adamstirtan.net):
- npm install
- npm run dev           # start Vite dev server (default port: 5173)
- npm run build         # build production assets (output in dist/)
- npm run preview       # optional: serve built dist locally via vite preview

Local static preview (simple):
- After `npm run build`, serve dist/ with a static server:
  python3 -m http.server 5173 --directory dist
  then open http://localhost:5173/

Production / Deployment notes
----------------------------
This project historically expects the built assets to be available at /assets/ (index.html references /assets/index-*.js and /assets/index-*.css). When deploying to static hosts (Azure, Netlify, Vercel, GitHub Pages), ensure one of the following:
- Deploy the contents of dist/ so the host serves /assets/* from the webroot, or
- Place the dist/ assets into an assets/ directory at repo root (this repo contains assets/ for straightforward hosting). 

If you change the Vite build output names or base path, make sure index.html matches the final asset filenames.

Troubleshooting
---------------
Symptom: Blank white page, title present
- Cause: index.html served but referenced /assets/* files return 404 (missing built assets).
- Fixes applied previously:
  - Replaced root index.html with the production-built index (dist/index.html) so it references correct asset names.
  - Copied dist/assets/* and dist/favicon.png into repo root (assets/ and favicon.png) so static hosts that serve repository files will expose the built assets.

How to debug locally
- Use Playwright (or open Chrome) and watch console errors; 404s for /assets/* indicate deployment mismatch.
- Serve dist/ locally (python3 -m http.server) and confirm assets load.

The "glitch" feature
---------------------
Location: src/components/Terminal.vue
Behavior:
- Intermittent short glitches are scheduled on mount. Scheduling: next glitch after 3.5s + random(0..8.5s) (avg ~7.8s).
- Glitches are light or heavy (45% heavy) with durations ~70–180ms (light) or ~160–420ms (heavy). 35% chance of quick aftershock.
- During a glitch characters are visually corrupted (random glyphs or ASCII substitutions) and visual effects (scanline, skew, shake, chromatic aberration) are applied.
- A click/tap on the terminal triggers a long heavy glitch (0.8–2.0s) immediately.

Toggle / Gate options
- Recommended: gate the effect behind a query param (eg. ?hack=1) or an environment flag so production default is no glitch. File: Terminal.vue — you can implement a simple check using URLSearchParams.
